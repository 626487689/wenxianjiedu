# 文献解读系统执行计划（2026-04-10）

关联需求：`docs/requirements/2026-04-10-wenxianjiedu-stability-and-batch-optimization.md`

## 1. 内部 grade 决策

- Grade：`L`
- 原因：本次是中等复杂度的单项目串行改造，模块有关联，适合在同一受治理执行道中分阶段实现，而不是 XL 并行拆分。

## 2. 执行波次

### Wave 1：状态与 bug 修复

范围：
- 批处理取消语义
- 任务状态统计一致性
- 错误分类与展示

输出：
- 明确的任务结束态
- 取消不再计入失败
- UI 错误提示更清晰

### Wave 2：模型适配增强

范围：
- 模型配置结构扩展
- endpoint 归一化策略改造
- 请求/响应兼容逻辑增强
- 连接测试反馈增强

输出：
- 更稳的 OpenAI-compatible 模型接入层

### Wave 3：批处理增强

范围：
- 并发数
- 重试次数
- 跳过已有输出
- 批处理汇总结果

输出：
- 更可用的批处理执行器与状态展示

### Wave 4：AI 稳定性增强

范围：
- 长文汇总前重试/缩减/降级路径
- 分块与阶段展示优化

输出：
- 长文场景下更稳的结果生成流程

## 3. 所有权边界

1. `src/types/*`
   - 负责新增配置与状态类型。
2. `src/ui/*`
   - 负责配置输入、批处理参数展示、错误与状态呈现。
3. `src/app/*`
   - 负责 facade 与依赖装配扩展。
4. `src/usecases/*`
   - 负责核心业务语义、批处理执行、重试与取消逻辑。
5. `src/services/llm/*` 与 `src/utils/*`
   - 负责 endpoint 与响应兼容策略。

## 4. 实施步骤

1. 先梳理现有任务状态、错误传播与批处理链路。
2. 完成取消语义和状态统计修复。
3. 扩展模型配置类型与 UI。
4. 调整 endpoint 归一化与请求解析逻辑。
5. 改造批处理执行器，加入并发/重试/跳过策略。
6. 增加批处理结果汇总展示或报告。
7. 增强长文汇总失败恢复策略。
8. 运行构建与手动验证。

## 5. 验证命令

在项目目录执行：

1. `npm run build`

必要时补充手动运行验证：

1. 单文件解读
2. 批量处理 + 取消
3. 错误 endpoint / 错误 key / timeout
4. 长文分块与汇总失败降级

## 6. 交付验收计划

1. 先验证不会破坏当前单篇流程。
2. 再验证批处理增强项是否符合需求文档。
3. 最后验证模型适配与长文稳健性改造。

## 7. 回滚规则

1. 若模型适配改造导致原有兼容服务不可用，优先保留原默认兼容路径。
2. 若批处理并发带来状态错乱，优先退回到串行但保留配置结构。
3. 若汇总增强影响输出稳定性，优先保留已有降级输出链路。

## 8. phase cleanup 预期

1. 不保留临时调试代码。
2. 删除未使用导入与废弃分支。
3. 记录验证结果。
4. 仅在验证完成后使用完成性措辞。
