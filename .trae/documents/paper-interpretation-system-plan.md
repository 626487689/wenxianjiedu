# 论文解读系统实现计划

## 1. 项目现状分析

### 1.1 当前系统功能
- 支持从本地文件或文件夹加载论文（PDF、MD、TXT）
- 支持配置OpenAI兼容的大模型API
- 支持批量处理和单个文件处理
- 支持自定义提示词模板
- 输出结果为Markdown文件
- 基于React+TypeScript+Tauri构建的桌面应用

### 1.2 现有技术栈
- 前端：React 18.3.1, TypeScript 5.6.2
- 构建工具：Vite 5.4.8
- 桌面应用：Tauri 2.10.1
- PDF处理：pdfjs-dist 5.6.205
- 文件系统：@tauri-apps/plugin-fs
- 对话框：@tauri-apps/plugin-dialog

## 2. 新功能需求

### 2.1 核心功能
1. **Zotero集成**：从Zotero获取论文元数据
2. **论文方向分析**：根据论文内容分析研究方向
3. **分段分析**：按照最大token进行智能分段
4. **多模型适配**：支持多种大模型API
5. **Obsidian兼容**：输出格式适配Obsidian
6. **本地运行**：确保所有处理在本地完成
7. **图形界面**：优化用户界面体验

## 5. 技术实现细节

### 5.1 Zotero集成
- 使用Zotero的SQLite数据库（默认路径：`%APPDATA%\Zotero\Zotero\Profiles\<profile>\zotero.sqlite`）
- 实现元数据读取和导出功能
- 支持选择特定集合或标签的论文

### 5.2 论文方向分析
- 使用LLM分析论文摘要、关键词和引言
- 生成方向标签和分类
- 支持自定义分析维度

### 5.3 智能分段
- 基于模型的max_token限制计算分段
- 保持段落和章节的完整性
- 支持重叠分段以确保上下文连贯性

### 5.4 多模型适配
- 支持OpenAI、Anthropic、Google等主流模型
- 实现模型能力自动检测
- 优化不同模型的提示词格式

### 5.5 Obsidian兼容
- 生成符合Obsidian规范的Markdown
- 添加Frontmatter元数据
- 支持Obsidian链接和标签系统

### 7.1 风险：Zotero数据访问权限
- 解决方案：提供Zotero数据库路径配置，确保用户授权

### 7.2 风险：不同模型API兼容性
- 解决方案：实现适配器模式，统一API调用接口

### 7.3 风险：分段处理性能
- 解决方案：优化分段算法，使用Web Workers处理大文件

### 7.4 风险：本地资源占用
- 解决方案：实现资源使用监控，优化内存管理
